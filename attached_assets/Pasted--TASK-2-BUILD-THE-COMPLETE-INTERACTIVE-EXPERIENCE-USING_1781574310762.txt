# TASK 2 – BUILD THE COMPLETE INTERACTIVE EXPERIENCE USING THE EXISTING MASCOT

The mascot already exists.

Do NOT redesign the mascot.

Do NOT generate a new character.

Use the existing mascot as the central guide throughout the entire experience.

The mascot must appear in every step and every interaction.

---

# PROJECT GOAL

Create a premium interactive onboarding experience for a vehicle tracking and fleet management company.

This should feel like:

* Duolingo onboarding
* Apple product experience
* Stripe onboarding
* Modern SaaS product tour

It must NOT feel like a form.

Users should feel like they are having an interactive conversation with the mascot.

---

# USER EXPERIENCE RULES

Only one question is visible at a time.

No long forms.

No multiple sections.

No overwhelming UI.

Each screen should focus on a single decision.

Large buttons.

Large typography.

Generous spacing.

Smooth transitions.

---

# LANDING EXPERIENCE

Page loads.

Mascot enters the screen.

Mascot waves.

Mascot smiles.

Speech bubble appears.

"Merhaba 👋"

"Size en uygun araç takip çözümünü birlikte bulacağız."

Continue button:

"Hadi Başlayalım"

---

# PROGRESS SYSTEM

Display a premium progress bar.

Progress updates after every answer.

Examples:

20%
40%

80%
100%

Progress animation must be smooth.

Every answer triggers a unique mascot reaction.

Examples:

Mascot becomes excited.

Small fleet:
Mascot gives thumbs up.

Premium option:

Basic option:

Every answer should feel acknowledged.

Everything should feel premium.
